# GramGPT v0.2 — Структура проекта

## 📁 Файлы и их роли

```
gramgpt/
│
├── main.py              ← точка входа. Только меню и вызовы других модулей
├── config.py            ← все настройки (API ключи, пути, константы)
├── db.py                ← база данных (сохранение/загрузка JSON)
├── tg_client.py         ← работа с Telegram (авторизация, проверка, спамблок)
├── proxy_manager.py     ← всё про прокси (загрузка, проверка, назначение)
├── trust.py             ← расчёт Trust Score и рекомендации
├── ui.py                ← весь вывод в терминал (цвета, карточки, меню)
│
├── data/
│   ├── accounts.json        ← база аккаунтов (создаётся автоматически)
│   ├── accounts_export.json ← экспорт
│   └── proxies.json         ← база прокси
│
├── sessions/
│   └── *.session            ← сессии Telegram (по одному файлу на аккаунт)
│
├── logs/                    ← логи (зарезервировано для следующих версий)
│
└── proxies.txt              ← (опционально) файл со списком прокси для импорта
```

---

## 🚀 Быстрый старт

### 1. Установи зависимости
```bash
pip install telethon colorama PySocks
```

### 2. Получи API ключи
1. Зайди на https://my.telegram.org
2. API development tools → создай приложение
3. Скопируй `api_id` и `api_hash`

### 3. Вставь ключи в config.py
```python
API_ID = 12345678
API_HASH = "твой_хэш_здесь"
```

### 4. Запусти
```bash
python main.py
```

## 📋 Функционал v0.2

| Модуль | Что делает |
|--------|-----------|
| Авторизация | Вход по номеру + SMS, поддержка 2FA |
| Проверка | Статус (жив/заморожен/спамблок) через @SpamBot |
| Trust Score | Балл 0–100 с рекомендациями |
| Прокси | Загрузка из файла, проверка, назначение на аккаунты |
| База данных | Всё сохраняется в data/*.json |
| Экспорт | Выгрузка аккаунтов в JSON |

## 📄 Формат файла proxies.txt

По одному прокси на строку, любой из форматов:
```
socks5://login:pass@1.2.3.4:1080
1.2.3.4:1080:login:pass
1.2.3.4:1080
```
